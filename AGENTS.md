# Repository Instructions

Keep changes small, reviewable, and covered by the local checks in `README.md`.

Do not include credentials, private paths, private repository data, or local runtime artifacts in examples, tests, issues, or pull requests.

Shell, filesystem write, edit, and network tool use must follow the repository policy and require approval when the policy says so.

Destructive actions, force pushes, and public artifact publication require explicit maintainer approval.

Run `pytest` and the demo guard checks before reporting a change as complete.
